# 第 5 章：讓秘書記住你的規則

Claude Code 其實有記憶功能，我們糾正過它的事、教過它的偏好，它會自己記下來，下次對話還記得。但這種記憶是秘書自己做的筆記，它覺得重要的才會記，有時記得很好，有時就漏掉了。像是「報價單格式要用公司模板、金額四捨五入到整數、最後一欄放備註」這種有三四個條件的規則，記住兩個漏掉一個是常有的事。要讓秘書每次上班都照同一套標準做事，光靠它自己的筆記不夠穩，我們得給它一份白紙黑字的工作守則 SOP。

## 5.1 CLAUDE.md 讓秘書每次都記得規則

### CLAUDE.md 是什麼？

`CLAUDE.md` 是一個純文字檔案，放在工作資料夾裡。例如我們一直在 `AI Space` 操作 Claude，就可以在這邊放一個 `CLAUDE.md`，Claude Code 每次啟動時，會自動讀取這個檔案的內容，當作「行為準則」。

新員工第一天上班，給他一本工作守則：「檔案命名用這個格式、所有文件用繁體中文、整理完的東西放到指定資料夾。」有了這本手冊，不管哪天上班，產出都是一樣的。

### 動手做：建立第一份 CLAUDE.md

開一個新對話，選 AI Space 資料夾，然後輸入：

```
在目前資料夾建立 CLAUDE.md，內容如下：

# 工作規則

## 語言
- 一律用繁體中文回覆
- 遇到英文專有名詞（像 Python、Claude Code），前後跟中文之間加空格

## 檔案管理
- 腳本存到 tools/ 資料夾
- 產出的檔案用「日期_描述」格式命名，例如 20260323_會議記錄.md
- 不確定存到哪裡就先問我

## 安全
- 刪除檔案前一定要先列清單讓我確認
- 不要自己安裝新套件，先告訴我需要什麼、為什麼需要
- 不要讀取或修改密碼、金鑰相關的檔案

## 做事方式
- 不確定的事情問我，不要自己猜
- 操作前先說你打算怎麼做，我同意再動手
- 一次做一件事，做完再問下一步
```

後面章節如果裝了新工具（例如 ffmpeg、whisper），記得回來把它加進「可用的工具」那一行，這樣 AI 就知道可以直接用，不會每次都嘗試重新安裝。

寫 CLAUDE.md 的時候跟其他對話一樣，指示必須具體。「格式好看一點」這種描述等於沒寫，「檔名用日期_描述格式」這種才有用。

建完之後，資料夾結構會像這樣：

```
AI Space/
├── CLAUDE.md          ← 秘書的工作守則
├── 測試素材/          ← 測試資料
└── tools/             ← 工具腳本
```

### 工作守則別寫成百科全書

有些人喜歡把 `CLAUDE.md` 當成作文寫，寫出幾百行的超長規則，想展現自己想得多周到，可惜 AI 讀完之後，因為規則太多了，有時候漏了忘了，有時候規則互相打架，反而表現更差。

經驗法則：一份 `CLAUDE.md` 建議控制在 80 行以內，系統其實沒有硬性限制，只是寫太多出錯的可能性會增加。所以我們寫一些最重要的原則跟常用的步驟就好，執行重複性高的特定任務時，使用下一節的 Skill；如果某個規則是臨時需要，直接在對話時說就好。

### 品質三板斧

Claude Code 開發團隊自己內部有三招提升產出品質，照順序來：

1. **開深度思考**：直接在對話框跟秘書說「幫我把 thinking 設成 always」，它會自己去改設定。開了之後 AI 回答前會先想清楚再動手，不是看到指令就衝。
2. **把 CLAUDE.md 寫好**：規則越明確，AI 越不會自作主張。
3. **讓 AI 自己驗證結果**：叫它跑完腳本之後自己檢查輸出對不對，或是拿產出跟原始資料比對。不要寫完指令就盲信結果，跟不校對就交報告一樣危險。

三件事都不難，但很多人連第一件都沒做，就在抱怨 AI 不好用。

### CLAUDE.md 放哪裡？

Claude Code 會從兩個層級找 `CLAUDE.md`：

1. **目前開啟的資料夾**（專案級）：只對這個資料夾有效。適合放專案特定的規則，例如「這個專案的報告要用英文」。也可以在子資料夾裡放更細的 `CLAUDE.md`，只會對那個子資料夾生效。
2. **`~/.claude/CLAUDE.md`**（全域級）：`~` 就是使用者資料夾（Mac 是 `/Users/你的名字/`，Windows 是 `C:\Users\你的名字\`），放在這裡的規則對所有專案都有效。適合放通用規則，例如「所有檔案用繁體中文」。

如果兩個地方都有 `CLAUDE.md`，Claude Code 會全部讀取，全部遵守。越接近工作資料夾的規則優先級越高（專案級蓋過全域級）。實務上建議不要讓規則互相矛盾，把通用的放全域、專案特定的放專案級，分清楚最省事，不過這邊我們就不展開說了。



## 5.2 Skill 讓秘書跟著 SOP 執行

CLAUDE.md 是通用規則，像「上班要穿制服、講話要有禮貌」。但有些任務比較複雜，需要一整套 SOP。比如整理照片這件事，前面我們是手動跟秘書講「模糊的刪、重複的刪、被遮擋的刪」，但如果每次都要重新交代一遍判斷標準，不是很蠢嗎？

把這些標準寫成 Skill，以後秘書自己就知道怎麼判斷。

### Skill 我知道，就是技能嘛！

你說的沒錯，不過在 Claude 裡面，Skill 比較像是一份詳細的 SOP 文件，把流程跟步驟寫好，包括什麼時候調用什麼工具。建好之後，Claude Code 需要執行這個流程時，會自己去讀對應的 Skill 檔案，照著步驟做。

跟 CLAUDE.md 的差別：

- **CLAUDE.md**：每次都會自動讀取的通用規則（工作守則）
- **Skill**：特定任務的 SOP，需要時才會被觸發（標準作業程序）

### 動手做：建立一個「照片整理」Skill

前面為了整理照片， Claude 做了一支 `tools/photo_cleanup.py` 的程式腳本。但如果每次要清理照片的時候，都要自己記得腳本放在哪裡、如何執行它，也是有點麻煩對吧，這種事情就可以做成一個 SKill 讓我們的 AI 秘書知道整理照片的時候應該直接跑這個腳本。

直接跟 Claude Code 說要建立Skill：

```
幫我建立一個照片整理的 Skill，當我說「整理照片」「清理照片」的時候自動觸發。觸發之後去跑 tools/photo_cleanup.py，目標資料夾由我指定。跑完之後列出結果，等我確認才能刪
```

Claude Code 會建好 Skill 檔案，格式它自己會處理。Skill 建好之後，它會自動變成一個 `/photo-cleanup` 的快捷指令，不需要額外設定。不過快捷指令建好之後，要**開一個新的對話**才會生效，在當前對話裡打 `/photo-cleanup` 是找不到的。

### 看看秘書寫了什麼

好奇秘書到底寫了什麼 SOP？在同一個對話裡直接說：

```
幫我打開技能檔
```

它會顯示剛才建的 Skill 檔案內容。打開來長這樣（節錄）：

```markdown
---
name: photo-cleanup
description: 掃描指定資料夾中的照片，找出重複、失焦、被遮擋的問題照片，確認後才刪除。當使用者說「整理照片」或「清理照片」時觸發。
---

# photo-cleanup skill

## 執行流程
### 1. 確認目標資料夾
- 如果使用者已指定資料夾路徑，確認路徑存在後繼續。
- 如果使用者未指定，必須先詢問目標資料夾路徑，不要猜測。

### 2. 執行掃描
用 Bash 執行掃描腳本...（省略具體程式碼）

### 3. 列出結果等待確認
重要：必須等使用者明確回覆後才能進行下一步。絕對不能自動刪除。

### 4. 執行刪除
優先移到垃圾桶，而非永久刪除。

### 5. 回報結果
刪除了哪些檔案、保留了哪些、檔案已移至垃圾桶（可復原）。
```

開頭那段 `---` 包起來的區塊叫 frontmatter，`name` 是 Skill 的名稱、`description` 告訴 AI 這個 Skill 是幹嘛的，觸發條件也寫在 `description` 裡面。下面就是完整的 SOP，從確認資料夾到掃描到等待確認再刪，每一步都寫得很清楚。

你只說了一句「幫我建照片整理的 Skill」，它自己就寫出了一百多行的流程文件，連「不能自動刪除」「優先用垃圾桶」這種安全機制都自己加上去了。上面是我這邊產出的範例，你建出來的內容可能跟這個不一樣，每次 AI 對話產出的結構和細節都會有差異，這都很正常。

如果想調整這個 Skill 的工作流程，除了像之前叫 AI 幫忙生成之外，也可以自己打開 Skill 檔案來編輯調整。

### 實測：觸發 /photo-cleanup

先把之前被刪除的測試圖檔還原，開新對話，在對話框打 `/photo-cleanup`，它會跳出來讓你選，點下去直接開始跑。秘書會問你要掃描哪個資料夾，可能是跳彈窗讓你選，也可能直接在對話裡問，每次不一定一樣。不用打完整路徑，打 `ch02_images` 它就會自己去找。找到之後開始掃描，跑的就是前面那支 `photo_cleanup.py`，幾秒鐘後列出結果：

```
掃描完成，找到 4 張有問題的照片：

| # | 檔名       | 問題                                           |
|---|-----------|------------------------------------------------|
| 1 | image3.jpg | 失焦 (模糊分數: 57.0)、近似重複 (與 image1 相似度 98%) |
| 2 | image4.jpg | 被遮擋 (鮮豔色塊共佔 24% 面積)                     |
| 3 | image5.jpg | 被遮擋 (鮮豔色塊共佔 41% 面積)                     |
| 4 | image2.jpg | 近似重複 (與 image1.jpg 相似度 98%)                |

請確認要刪除哪些照片（輸入編號，例如「1,3」或「全部」），
或說「取消」放棄操作。檔案會移到垃圾桶，可以復原。
```

跟前面讓 AI 看照片判斷的結果一樣，但這次是腳本在跑，不是 AI 逐張看圖。不用記路徑、不用記指令，打 `/photo-cleanup` 就搞定，最重要的是消耗的token減少了非常非常多。

自然語言也能觸發，不打 `/` ，直接說「幫我整理 ch02_images 的照片」，效果完全一樣。

### 怎麼確認秘書真的有跑腳本？

你可能會好奇：秘書列出來的結果，到底是它真的跑了 `photo_cleanup.py`，還是它自己看照片判斷的？看它的操作紀錄就知道。每次它用 Bash 工具執行指令，都會顯示它跑了什麼。你會看到它確實呼叫了 `python3` 去跑你的腳本，輸出裡那些模糊分數、相似度百分比都是腳本算出來的，不是 AI 自己編的。

下一節來玩一個更實用的：把好幾件事串成一個快捷指令。


## 5.3 快捷指令的組合技

`/photo-cleanup` 只做一件事，但快捷指令真正好用的地方是組合技：一個指令串起好幾個任務。

### 動手做：建立一個 /morning 指令

每天早上都想讓 Claude Code 做三件事：查日圓匯率、看星巴克有沒有特價、給你一句學習提醒。三件事分開講要打三段指令，不如包成一個 `/morning`，一次搞定。

直接跟 Claude Code 說：

```
幫我建一個 /morning 的快捷指令，每天早上打 /morning 就依序做這三件事：
1. 執行/jpy_rate_tracker.py 抓日圓匯率，腳本會自動跟近 30 天歷史比較並給分析報告
2. 上網查星巴克有沒有買一送一或其他優惠活動
3. 隨機給我一句今天的學習目標或正能量提醒

```

Skill 建好之後就自動變成 `/morning` 快捷指令，一樣要開新對話才會生效。

建好之後，每天早上只需要在 Claude Code 裡打 `/morning`，就可以看到AI一口氣執行三件任務，是不是在想，還要跟ai說morning多麻煩，不是應該早上九點我一開機他就自己來跟我說早安嗎？ 肯定可以的，別急，這個我們後面再加上．


### 帶參數的快捷指令

前面 `/photo-cleanup 圖片/三月出遊` 已經示範過了：指令後面加路徑，Claude Code 就知道要整理哪個資料夾。這種「每次輸入不同」的需求很常見，再來一個例子。

假設你常常需要摘要各種檔案：

```
幫我建一個 /summarize 的快捷指令，後面接檔案路徑，讀取那個檔案然後用繁體中文產出三段式摘要（背景、重點、結論），每段不超過 100 字
```

使用指令就可以快速做摘要：

```
/summarize report_q1.pdf
```


### 兩種觸發方式

每個 Skill 都有兩種觸發方式：

- **打 `/名稱`**：主動觸發，像按快捷鍵。打 `/photo-cleanup` 就跑照片整理，打 `/morning` 就跑早晨三件套
- **自然對話**：Claude Code 根據你說的話自動判斷。你說「整理照片」，它自己會去找照片整理的 Skill 來用

同一個 Skill，兩種觸發方式都能用，看你當下怎麼方便。

## 5.4 Memory：秘書自己的筆記本

CLAUDE.md 和 Skill 都是你主動寫的規則。Memory 不一樣，它是秘書自己做的筆記。

比如糾正過它一次「不要自己裝套件，先跟我說」，它會自己記下來，下次遇到同樣的情況就不會再犯。你也可以主動叫它記東西：「記住，我的日圓買入成本是 0.2185」，它就會存起來，以後聊到日圓相關的話題自動帶入。

這些記憶存在 `~/.claude/` 底下的記憶資料夾裡，就是普通的文字檔。可以直接打開來看秘書到底記了什麼，覺得哪條記錯了就刪掉或改掉。也可以在對話裡跟它說「忘掉 XXX」，它會自己去刪。

除了對話中糾正讓 AI 自己記之外，還有個更快的方式：在對話框輸入 `#` 會切換到記憶模式，打的內容直接存成長期記憶，不會被當成一般對話處理。想到什麼隨手 `#` 存一條，比對話中說「記住 XXX」更俐落。順便提另一個實用快捷鍵 `!`，按下去會切換到命令列模式，可以直接打 shell 指令，執行結果會自動加入對話上下文。想快速確認某個檔案在不在、看一下資料夾結構，按 `!` 打 `ls` 比跟 AI 說「幫我看一下」快得多。

Memory 我們先不需要去管它，讓 AI 自己維護就好。你糾正它、教它新東西，它自己會判斷哪些值得記下來。CLAUDE.md 管規則、Skill 管流程，這兩個才是要花心思維護的。Memory 就讓秘書自己做筆記，想看它記了什麼，直接在對話裡問「你記了哪些東西」就好。
