# 第 4 章：跟秘書高效溝通

到目前為止，秘書整理了下載資料夾、監控匯率並且給我們建議、刪除了幾張照片，已經充分驗證它的能力，但接下來我們會碰到新的問題，第一個是前面說掃描大量照片會出現 Token 不夠的問題，其實也不只是掃描照片，AI 每次讀取、思考、輸出都是要消耗 Token 的，例如說讓它一口氣讀完哈利波特全集，寫一份 300 萬字的寶可夢百科全書，都會遇到 Token 不夠的問題；另外一個是失憶症，當你打開一個新對話之後，秘書可能會把之前做過的事全忘了。或是說說叫它監控匯率，它又重新寫了一次腳本、怎樣的照片應該被刪除也忘了，每次都要重新交代。


## 4.1 省 Token 撇步 1：把指令說清楚

其實我們前面的示範沒有很好，指令下的沒有很精確，但也可以看出筆者下的指令還是比一般的口語對話還要長一點，主要目的除了讓 AI 不會出錯外，也可以節省它 Token 的消耗。

### 4.1.1 指令具體 > 指令模糊

這是最重要的原則，下達的指令越清楚，AI 去分析猜測意圖的消耗就越少，當然我們說得越清楚，老闆當的就越辛苦，具體怎麼平衡，只能說多用自然會找到平衡點。

**模糊的指令：**
```
幫我整理一下這些檔案
```

AI 會先去思考：這些是哪些？整理是什麼意思？按名稱排序？按日期分類？按類型放進不同資料夾？刪掉重複的？重新命名？全部都做？光是搞清楚你要什麼，就燒掉一堆 token。

**前面章節我們下的指令：**
```
幫我整理 測試素材/ch02_downloads 資料夾。把檔案按類型分到子資料夾，
然後看一下截圖的內容，幫它們取有意義的檔名。
先列出你打算怎麼改讓我確認，不要直接改動。
```

比模糊版好很多，指定了路徑、說明了怎麼整理、還加了安全閥「不要直接改動」。但這樣還可以更好：

**更精確的版本：**
```
幫我整理 測試素材/ch02_downloads 資料夾：
1. 把檔案按副檔名分到子資料夾（pdf/、jpg/、docx/）
2. 截圖類的圖片，讀取內容後取有意義的檔名，格式：日期_描述.png
3. 其他類型的檔案不要動
4. 先列出你打算怎麼改讓我確認，不要直接改動
```

雖然說更精確的版本會讓 AI 執行上更有效率，不過就如上面所說，打字打那麼多，到底誰才是老闆誰才是秘書，筆者實際使用上，是不可能輸入這種長度的指令，每個人習慣的方式不同，就只能靠讀者自己摸索了。


### 4.1.2 給範例 > 純描述

有些事情很難用文字精確描述，但給一個範例就一目了然。

**純描述：**
```
幫我把客戶名單整理成比較好看的格式
```
AI 會先去思考：「比較好看」是什麼意思？表格？條列？一行一筆？欄位要哪些？排序方式？AI 不知道你心裡想的「好看」長什麼樣子，它只能猜，猜錯了又要打槍重來，token 就多燒了一輪。

**給範例：**
```
幫我把客戶名單整理成以下格式：

名單範例：永遠排隊小籠包
聯絡人：又改需求的小許
電話：02-1234-5678
最後聯絡日期：2026-03-15
備註：Q2 要續約

請按「最後聯絡日期」由近到遠排序。
```

這個直接給範例的版本，直接把實際使用的資料格式給它看。AI 看到範例就知道：用冒號分隔、一個欄位一行、要有備註欄、按日期排序。不需要用文字解釋「表格式、有分隔線、要有備註欄」這些抽象描述，一個範例就全部講清楚了。

這招在整理任何有固定格式的東西時都很好用，會議記錄、通訊錄、帳單明細，只要貼一筆「長這樣」的範例，AI 就會照著格式處理剩下的全部資料。

### 4.1.3 一次一件事 > 一次塞十件

一次塞太多任務，AI 容易在中間某個步驟出錯，後面全部歪掉。更不幸的是，它會自己用新的方法重試，我們很難判斷到底是在哪一步出了問題。

如果我們還不確定要怎麼做，自然的做法是：**一次講一件事，看完結果再決定下一步。** 聽起來很像跟前面提到的「指令要精確」有點衝突，但其實是兩回事，精確是每句話的品質，一次一件是每輪對話的範圍。「幫我看下`xxx`裡面的照片」這句話很短，但指令是精確的：目標明確（看照片）、範圍明確（`xxx`裡面的）。它不模糊，只是範圍小。

回想前面章節整理照片的過程。我們沒有一口氣說「幫我看照片、找出廢片、然後刪掉」，而是先說：

```
幫我`xxx`看下裡面的照片
```

秘書看完資料夾後回報每張照片的內容，我們確認它判斷得沒問題，才接著說：

```
幫我刪掉被遮擋、重複還有失焦的照片
```

如果第一步它就看錯了（比如把正常照片誤判為應該刪除的照片），我們馬上就能糾正，不用等它刪完才發現搞砸了。每一步都確認過再往下走，出錯的成本最低。

### 4.1.4 告訴它「不要做什麼」也很重要

AI 有時候非常熱心，會自己加料。比如你請它整理檔案，它順手幫你把覺得「不需要」的檔案刪了。你沒叫它刪，它自己覺得在幫你，但那個檔案可能是你寫到一半的論文，明天要上臺報告的投影片。

所以當你有明確禁止 AI 執行的動作時，例如說我們只是要 AI 做檔案分類，這樣說會比較安全：

```
把檔案按類型分類，但不要刪除任何檔案，不要修改檔名
```


### 4.1.5 指令太長就用編號列表

如果指令超過十行，AI 容易漏掉中間的細節。把一大段話拆成編號列表，效果會穩定很多。

不好的寫法：

```
幫我把桌面的檔案整理一下，PDF 放一個資料夾，圖片放一個資料夾，然後那些截圖要按日期重新命名，格式是年月日加底線加描述，如果有重複的檔案就保留比較新的那個，舊的移到一個叫 duplicates 的資料夾，對了 .DS_Store 那些系統檔不要動
```

好的寫法：

```
整理 ~/Desktop 的檔案，規則如下：

1. 按副檔名分類：PDF → pdf/，JPG/PNG → images/
2. images 資料夾裡的截圖按日期重新命名，格式：20260323_描述.png
3. 重複檔案：保留較新的，舊的移到 duplicates/
4. 不要動 .DS_Store 等系統隱藏檔
```

同樣的內容，拆成編號之後 AI 會逐條執行，不容易漏。這也是一種更精確的版本。

## 4.2 省 Token：一事一議，做完就開新對話

同一個對話聊久了，Claude Code 會開始恍神，而且特別吃 Token，這不是 bug，是它先天的運作方式。

### 4.2.1 為什麼 AI 會越聊越笨？

每次對話，Claude Code 會把之前的所有內容都記在它小小的「腦袋」裡，這個腦袋叫上下文視窗，有嚴格的容量上限。前幾輪對話它看起來聰明絕頂，回答又快又準。聊到一半的時候會覺得它記憶力真好，之前你提過早忘記的東西它都還記著，但當又聊了更多更久、做了很多任務之後，早期的對話開始被壓縮，很多資訊都在壓縮過程遺失了，它的理解力也跟著斷崖式下降。會發現這傢伙開始重複問已經回答過的問題，或者把十分鐘前才講好的規則忘得一乾二淨。

而且對話越長，每次交互消耗的 token 越多，因為它每次回覆都要把整段對話歷史重新讀一遍，token 也會越燒越快。不過也不用太擔心，Pro 方案的額度是以 5 小時為一個週期計算的，正常使用的話，像是整理檔案、寫腳本、問問題這些日常任務，一整個下午都用不完。會把額度燒光的通常是叫它一口氣讀大量圖片、處理超長文件，或是在同一個對話裡聊了幾十輪不開新對話。只要養成做完一件事就開新對話的習慣，大部分人根本不會碰到上限。

### 4.2.2 「一事一議」原則

解法很簡單：做完一件事，開一個新對話。

- 整理完會議記錄 → 開新對話
- 接著要處理照片分類 → 新的對話
- 處理完照片要寫信 → 又一個新的對話

不要在同一個對話裡塞個不相關的任務。每個任務都從乾淨的狀態開始，AI 的表現會穩定，token 消耗也少。

### 4.2.3 什麼時候可以留在同一個對話？

如果兩件事有強相關，留在同一個對話反而比較好。比如：

- 先整理逐字稿，再根據整理結果寫會議記錄
- 先分析一份報告，再根據分析結果寫摘要

因為後面的任務需要用到前面的結果。如果開新對話，它就不知道前面做了什麼，你又要重新餵一次資料。

判斷標準：**後面的任務有沒有用到前面的產出？** 有，就留在同一個對話。沒有，就開新的。

## 4.3 省 Token：用 .claudeignore 過濾雜訊

前面提過 `.claudeignore`，當時是從安全性的角度講：把不想讓 AI 碰的檔案列進去。但它還有另一個好處：省 token。

Claude Code 在需要找檔案或了解資料夾結構的時候，會掃描工作資料夾。如果資料夾裡有幾千個檔案，掃描就要花不少時間和 token。把跟當前任務無關的東西排除掉，秘書就不會浪費力氣去讀那些「雜訊」。

在你的工作資料夾建立 `.claudeignore`（如果前面章節已經建過，就打開來加東西），實際上要加什麼東西，也可以諮詢 AI 秘書討論怎麼做比較好。這邊先給一個範例，可以上 GitHub 直接複製貼上：

```
# 系統垃圾
.DS_Store
Thumbs.db
desktop.ini

# 暫存檔
*.tmp
*.cache
~$*

# 大型媒體檔（不需要 AI 處理的）
*.mp4
*.mov
*.avi

# node_modules（如果你有在用網頁相關的工具）
node_modules/

# 舊備份
backup/
old/
```

### 奇怪，AI 怎麼不去讀某個檔案

最常見的意外是用了太廣泛的萬用字元，比如 `*.md` 本來是要排除草稿筆記，結果把正式文件也擋掉了。遇到 AI 說找不到某個確定存在的檔案時，第一件事就是去檢查 `.claudeignore`，或是讓 AI 秘書去幫忙看看。

## 4.4 腳本重複利用，秘書不用什麼事情都自己來

前面三招都是在「省著用」，這一招是真正的大絕：**把重複的工作交給腳本，徹底不消耗 token。**

回想一下前面提過的匯率腳本。Claude Code 幫寫了一支 `jpy_tracker.py`，抓日圓歷史紀錄、即時匯率、買進建議。如果隔天又打開 Claude Code 跟它說「幫我查一下今天日圓匯率」，它不去執行昨天寫好的那支腳本，而是重新理解需求、重新找資料來源、重新跑一遍整個流程。雖然最終還是能完成任務，但 token 白白燒掉了。

正確做法是告訴它去跑現成的腳本：

```
幫我用 jpy_tracker 補一下日圓歷史資料
```

這樣 Claude Code 只會執行一行指令，不會重新發明輪子。消耗的 token 只有這句對話，比「幫我查日圓匯率」從頭來過省了 90% 以上。

如果連這點 token 都想省，直接在對話框輸入 `!python3 jpy_tracker.py`，驚嘆號開頭的指令會直接執行，不經過 AI 思考，零 token 消耗。

### 4.4.1 什麼時候才需要再找秘書？

腳本壞掉的時候。

比如匯率資料來源某天改版了，腳本跑出來一堆錯誤。這時候才需要把錯誤訊息貼給 Claude Code，請它修。修完之後，你又回到「自己跑腳本」的模式。

原則很簡單：

- **第一次做**：找秘書，讓它幫你把工具造出來
- **每天重複做**：叫秘書跑工具，自己跑工具
- **工具壞了**：找秘書修


### 處理大量照片

現在就來讓 AI 處理大量照片又不花太多 Token。點左邊欄找到之前刪照片的那個對話，點進去繼續。先讓它把照片還原，或是自己把整理前的照片放回 `ch02_images` 資料夾下面，接著下指令：

```
我要你寫一個腳本放到 AI Space 下面的 tools 資料夾，功能是整理照片，規則是找出被遮擋、重複還有失焦的照片，給我確認刪除
```

秘書收到之後會先去檢查電腦裡有哪些現成的工具可以用。它可能會跟你說「需要安裝 opencv-python 和 imagehash 套件」，聽起來很專業，但這時候你可以直接問一句：

```
需要嗎
```

別小看這三個字。秘書重新想了一下，發現電腦裡已經有替代工具可以處理，不用再安裝額外工具，用現有的就能搞定失焦偵測、重複比對、遮擋判斷。這就是當老闆的好處，員工說要買新設備，你多問一句「真的需要嗎」，有時候現有的工具就夠用了。

腳本寫好之後，跑起來會掃描資料夾裡的所有照片，然後列出有問題的：

```
📂 掃描 測試素材/ch02_images，共 5 張照片

找到 4 張有問題的照片

[1] image2.jpg
    → 近似重複（與 image1.jpg 相似度 98%）
[2] image3.jpg
    → 失焦（模糊分數: 57.0）
    → 近似重複（與 image1.jpg 相似度 98%）
[3] image4.jpg
    → 被遮擋（鮮豔色塊共佔 24% 面積）
[4] image5.jpg
    → 被遮擋（鮮豔色塊共佔 41% 面積）
```

跟前面章節讓 AI 逐張看照片的結果一樣，但這次是腳本自己跑的。5 張照片感覺不出差別，換成 500 張呢？

| 方式 | Token 用量 |
|------|-----------|
| 腳本掃描 | 大約 500 tokens（一次指令呼叫，回傳文字結果） |
| AI 逐張看 | 大約 750,000+ tokens（每張圖大約 1,500 tokens × 500 張） |

大約 1,500 倍的差距。而且 AI 一次讀不完 500 張，要分好幾批，實際消耗只會更多。腳本幾秒鐘跑完，AI 可能要跑半小時還把額度燒光。

以後要整理照片，只要在對話框跟秘書說「幫我跑 photo_cleanup.py，目標資料夾是 ~/Pictures/要整理的資料夾」，或是用驚嘆號直接執行：

```
!python3 tools/photo_cleanup.py ~/Pictures/要整理的資料夾
```

工具造一次，用一輩子。

### 找不到腳本？指定路徑就好

讓 Claude Code 寫完腳本後，隔天開新對話就找不到檔案了。

預防方法：在下指令時就指定路徑。

```
幫我寫一個 Python 腳本，存到 AI Space 的 tools 資料夾裡，檔名叫 jpy_tracker.py
```

養成習慣，讓所有腳本都集中在同一個資料夾。以後要跑什麼工具，去那個資料夾找就好。

## 4.5 幾個讓你事半功倍的操作技巧

前面四節都在講怎麼省 token，這節換個方向，聊幾個操作上的小撇步，知道了以後用起來會順很多。

### 1. 看到不對路，立刻按 Esc

你下完指令，秘書開始噼里啪啦地執行，結果一看方向完全不對。這時候不用客氣，直接狂按 `Esc` 就能中斷它。中斷之後告訴它哪裡搞錯了，它會從你糾正的地方重新來過。

很多人會想說「算了讓它跑完再說」，這是最虧的做法。它跑得越久，燒掉的 token 越多，而且跑完之後你還是得打回票重來。方向錯了就立刻喊停，省下來的 token 拿去做正事。

### 2. Plan Mode：先看計畫再動手

在輸入框按 `Shift+Tab` 可以切換模式，其中的計劃模式 Plan Mode 特別實用。開啟之後，秘書只會列出它打算怎麼做，不會真的動手。你看完計畫覺得沒問題，再切回一般模式讓它執行。這招對大任務特別有用。比如你要它「幫我把公司這三年的成果匯總並整理成報表」，與其讓它直接衝，不如先開 Plan Mode 看看它打算怎麼分類、用什麼格式、輸出到哪裡。花一分鐘對齊方向，省下十分鐘的冤枉路。小任務就不用了，殺雞不必動到牛刀。

### 3. 主動 Compact，不要等它自己來

對話累積到一定長度，Claude Code 會自動壓縮前面的內容，這個動作叫 compact。問題是它挑的時機不一定好，如果剛好在任務執行到一半壓縮，重要的細節可能在壓縮過程中被丟掉，接下來的準度就會下降。

比較好的做法是你自己來決定什麼時候壓縮。完成一個階段的任務後，看一下 context 用量，如果已經用到七八成，主動在輸入框打 `/compact` 讓它壓縮，或者乾脆開一個新對話。重點是讓壓縮發生在任務的「空檔」，而不是「半路上」。

### 4. 不懂的問題，丟給另一個 AI

用到一半秘書突然問：「你要用什麼資料庫？」「這個 API 的認證方式是什麼？」你心想我要是知道我還需要你嗎？

沒關係，把它問你的問題複製起來，貼到網頁版的 Claude 或 ChatGPT，跟另一個 AI 說「我在用 Claude Code 做某某事，它問我這個問題，幫我回答」。拿到答案之後貼回 Claude Code 繼續。不需要自己懂技術，只要會複製貼上就好。兩個 AI 互相搭配，你負責當中間的傳話人，這個組合比你想像的強大很多。

### 5. 一個動作大概花多少 Token？

Pro 方案有每日用量上限，但 Claude Code 不會告訴你「還剩多少」，只會在快用完的時候在右下角提醒你用量已經90%, 95% 99%, 然後就突然停下來，這時你就只能等額度恢復或是加錢買更高級的Max方案。通常一個簡單的任務，像是整理幾個檔案、寫一段文字，大約消耗一萬到一萬五千個 token。聽起來很多，但 Pro 方案的每日額度其實不算少，正常使用一整天綽綽有餘。

真正會出事的是同時開好幾個對話視窗，每個都在跑任務。這樣 token 消耗是倍數成長，不到一小時就可能撞到上限。建議同一時間只開一個對話做事，做完再開下一個，把額度花在刀口上。
