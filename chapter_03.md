# 第 3 章：安全性與權限

相信在剛才那一章，你已經按確認按到手軟了——怎麼一堆事情都要我確認？這完全不自動也不聰明啊。別緊張，這只是 Claude Code 出廠的預設狀態「什麼都先問」。每次要動檔案或跑指令之前，都會跳一個確認視窗，問要不要放行，這是一種保護機制，避免秘書把你家也給拆了。選項有三個：

- **Allow**：允許這一次，下次同類操作還是會問
- **Always allow**：這類操作以後都自動放行，不再煩人
- **Deny**：拒絕，它會停下來等新指示

看到它要跑一串完全看不懂的英文指令？不要慌，也不要隨便按同意。按 Deny，然後問它：「你剛想做什麼？」它會用白話解釋。聽完覺得合理，叫它重來；覺得不對勁，叫它換個方式。Claude Code 本身就是最好的翻譯官。

雖說是為了安全，一直按確認就飽了，還做不做事啊？花在按確認的時間比下指令還多。每次讀個檔案要按一下確認，上網搜尋也問，開個資料夾、寫個檔案通通都要問，這多麻煩。

沒事沒事，我們馬上就來處理這個問題。

## 選你的風格：黑名單 vs 白名單

Claude Code 的權限設定檔在 `~/.claude/settings.json`。先把它打開：直接在 Claude Code 的對話框裡說「幫我打開 ~/.claude/settings.json」，秘書會讀取這個檔案，回覆裡會出現一個可以點的 `settings.json` 連結，點下去就會在 VS Code 裡打開，直接編輯。

基本上可以分為兩種思路，黑名單派跟白名單派：

| | 黑名單派 | 白名單派 |
|---|---------|---------|
| 邏輯 | 預設全放行，只擋危險的 | 預設全擋，只開放安全的 |
| 適合 | 懶得按確認，電腦裡沒機密 | 對資安比較敏感的電腦 |
| 好處 | 最省事，幾乎不會被打斷 | 沒有驚喜，新指令都要點頭 |
| 壞處 | AI 用了沒想到的指令不會事先知道 | 被不在清單裡的指令打斷 |

### 黑名單派：預設全放行，只擋危險的

這條路適合「懶得每次按確認」而且「電腦裡沒什麼機密」的人，簡單說就是方便可是沒那麼安全。邏輯是：我信任這個秘書，讓它自由發揮，但明確告訴它哪些抽屜不準碰。

把下面這段 JSON 複製到 `~/.claude/settings.json` 裡面，GitHub 上也都有提供這些範例，可以直接複製使用：

```json
{
  "permissions": {
    "deny": [
      "Bash(rm -rf *)",
      "Bash(curl *)",
      "Bash(wget *)",
      "Bash(ssh *)",
      "Read(./.env)",
      "Read(**/.env)",
      "Read(~/.ssh/**)"
    ]
  }
}
```

稍微說明一下我們做了哪些限制：

- **`rm -rf`**：把整個資料夾連同子資料夾全部刪光，不問也不給反悔的機會
- **`curl`、`wget`**：能把你電腦裡的檔案透過網路傳到外面。秘書正常上網查資料不受影響（它有自己的上網工具，不需要用這兩個指令）
- **`ssh`**：能連到別的機器，在你不知情的情況下操作遠端電腦
- **`.env`**：這種檔案通常存放 API 金鑰和密碼，不該讓 AI 隨便讀
- **`.ssh/`**：你的 SSH 金鑰，等於電腦的身分證，外洩就能冒充你登入其他機器

除了 `deny` 清單裡列出來的，其他所有操作都會自動放行。

好處：最省事，日常使用幾乎不會被打斷。
壞處：AI 用了什麼指令，我們基本上不會事先知道。

### 白名單派：預設全擋，只開放安全的

這條路適合「電腦裡有客戶資料、公司機密」或者「寧可多按幾次確認也不要出事」的人。邏輯相反：秘書什麼都不準碰，我一項一項告訴你哪些可以用。

```json
{
  "permissions": {
    "allow": [
      "Read",
      "Edit",
      "Write",
      "Bash(git *)",
      "Bash(python3 *)",
      "Bash(node *)",
      "Bash(ls *)",
      "Bash(mkdir *)",
      "Bash(mv *)",
      "Bash(cp *)",
      "Bash(head *)",
      "Bash(tail *)",
      "Bash(cat *)",
      "Bash(grep *)",
      "Bash(find *)",
      "Bash(wc *)",
      "Bash(sort *)",
      "Bash(uniq *)",
      "Bash(chmod *)",
      "Bash(date *)",
      "Bash(echo *)"
    ]
  }
}
```

`allow` 清單裡面都是日常操作：讀寫檔案、Git 版本控制、Python 和 Node 執行、基本的資料夾瀏覽和檔案操作。這些指令不會把資料傳出去，也不會刪掉不該刪的東西。

不在清單裡的操作？Claude Code 每次都會跳確認。比如它想用 `curl` 去抓網頁資料，會停下來讓人看一眼，覺得 OK 再按 Allow。

好處：沒有驚喜，每個新指令都經過點頭。
壞處：偶爾會被不在清單裡的指令打斷工作流程。做的事比較多元的話，一開始會需要常常手動放行，再把常用的指令加進清單裡。

### 不想自己改設定檔？

直接跟 Claude Code 說就好。把上面其中一段 JSON 複製起來，然後跟它說：「幫我把 `~/.claude/settings.json` 改成這樣。」再把 JSON 貼給它，它會幫你改好。

或者更懶一點，直接講：「幫我把權限設定改成黑名單模式，擋掉 sudo、rm -rf、ssh 和所有上傳指令。」它看得懂。

## .claudeignore：連看都不讓它看

不管選黑名單還是白名單，都建議設一個 `.claudeignore`。

`settings.json` 管的是「能做什麼動作」，`.claudeignore` 管的是「能看到什麼檔案」。兩層防護，邏輯不同。就算白名單裡只開放 `Read`，它還是能讀到工作資料夾裡所有檔案的內容，包括那個存了 API 金鑰的 `.env`。

`.claudeignore` 是一個純文字檔案，放在工作資料夾裡（就是用 VS Code 打開的那個資料夾根目錄）。檔案裡列的東西，Claude Code 完全看不到，不讀、不改、假裝不存在。

在 VS Code 裡建立這個檔案：左邊的檔案列表空白處按右鍵，選 New File，輸入 `.claudeignore`（前面那個小數點別忘了）。裡面這樣寫：

```
# 密碼和金鑰
.env
*密碼*
*password*

# 財務資料
報稅/
銀行對帳單/

# 大型檔案（不是安全問題，是省 token）
*.mp4
*.mov
*.zip
```

一行一個規則，`#` 開頭是註解。`*` 是萬用字元，`*密碼*` 表示檔名裡只要包含「密碼」兩個字就擋。資料夾後面加 `/` 表示整個資料夾都不讓它進。

有一件事要提醒：`.claudeignore` 只認檔名規則，不認檔案內容。一個叫做「機密資料.txt」的檔案，照上面的設定，AI 還是會去讀這個檔案的，因為檔名沒有命中任何規則。

## Hook：需要確認的就跳通知

Claude Code 有一個 Hook 機制，可以在特定事件發生時自動執行一段指令。我們拿來做自動通知用，當 AI 需要你確認的時候，就跳一個通知給你，免得你滑手機忘了 AI 還在工作。

### 動手做：approve 時跳通知（Mac）

直接跟 Claude Code 說：

```
幫我在 settings.json 加一個 Hook，當你需要我確認的時候，用 osascript 在 Mac 跳一個通知提醒我，要有聲音
```

它會自動幫你把 Hook 設定寫進 `~/.claude/settings.json`，跟前面的權限設定合在一起，JSON 格式也不用你操心。

設好之後，每次 Claude Code 停下來等你按 approve，Mac 右上角就會跳一個通知氣泡，還會叮一聲。你在旁邊開著瀏覽器查資料、回 LINE 訊息，滑著手機的時候，聽到聲音就知道再切回去按確認就好。

### Windows 可以嗎？

Mac 的通知用的是 `osascript`，這個在 Windows 上沒有。Windows 要做到類似效果需要透過 PowerShell 的 toast notification，設定比較麻煩，而且不同版本的 Windows 行為不太一樣。如果你是 Windows 用戶，這段可以先跳過，後面章節教 Telegram 通知的時候會有跨平台的解法。

秘書的通行證設好了，敏感資料也鎖進保險箱，等你確認的時候還會叮一聲叫你回來。接下來要教秘書怎麼聽懂你的話，讓它做事更有效率。
