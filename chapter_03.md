# 第 3 章：門禁卡管理：安全性與權限

相信在剛才那一章，你已經按 Allow 按到手軟了：怎麼一堆事情都要我允許？這完全不自動也不聰明啊。別緊張，這只是 Claude Code 出廠的預設狀態「什麼都先問」。每次要動檔案或跑指令之前，都會跳一個權限視窗，問要不要放行，這是一種保護機制，避免秘書把你家也給拆了。選項有三個：

- **Allow**：允許這一次，下次同類操作還是會問
- **Always allow**：這類操作以後都自動放行，不再煩人
- **Deny**：拒絕，它會停下來等新指示

看到它要跑一串完全看不懂的英文指令？不要慌，也不要隨便按同意。按 Deny，然後問它：「你剛想做什麼？」它會用白話解釋。聽完覺得合理，叫它重來；覺得不對勁，叫它換個方式。Claude Code 本身就是最好的翻譯官。

雖說是為了安全，可是AI秘書的任務一直被中斷，一直等著我們 Allow ，我們花在按確認的時間比下指令還多。每次讀個檔案要按一下確認，上網搜尋也問，開個資料夾、寫個檔案通通都要問，這多麻煩。

沒事沒事，我們馬上就來處理這個問題。

## 選你的風格：黑名單 vs 白名單

Claude Code 的權限設定檔在 `~/.claude/settings.json`，其實我們也沒要打開它，讓 AI 自己處理就好。

基本上可以分為兩種思路，黑名單派跟白名單派：

| | 黑名單派 | 白名單派 |
|---|---------|---------|
| 邏輯 | 預設全放行，只擋危險的 | 預設全擋，只開放安全的 |
| 適合 | 懶得按確認，電腦裡沒機密 | 對資安比較敏感的電腦 |
| 好處 | 最省事，幾乎不會被打斷 | 沒有驚喜，新指令都要點頭 |
| 壞處 | AI 用了沒想到的指令不會事先知道 | 被不在清單裡的指令打斷 |

### 黑名單派：預設全放行，只擋危險的

這條路適合「懶得每次按確認」而且「電腦裡沒什麼機密」的使用者，簡單說就是方便可是沒那麼安全。邏輯是：我信任這個秘書，讓它自由發揮，但明確告訴它哪些抽屜不準開，哪些行為是被禁止的。

直接在對話框跟 Claude 說：

```
幫我設定權限 settings.json，禁止以下操作，其他全部自動放行：

刪除類：rm -rf、rm -fr、dd、mkfs
提權類：sudo、su
網路類：curl、wget、ssh、nc、ncat
危險操作：eval、chmod 777、git push --force、git reset --hard
機密檔案：讀取和修改 .env，讀取 .ssh、.aws、.azure、.kube 資料夾，讀取 .git-credentials、.npmrc、.pypirc，讀取所有 .key 和 .pem 檔案，用 cat、less、head、tail 讀取上面這些檔案也要擋，修改 .bashrc 和 .zshrc
```

這份清單參考了資安社群常見的 Claude Code 安全配置。秘書會幫你把設定寫進 `~/.claude/settings.json`，完全不用自己動手改檔案。

每個項目在做什麼：

| 類別 | 指令 | 為什麼要擋 |
|------|------|-----------|
| 刪除 | `rm -rf`、`dd`、`mkfs` | 刪除整個資料夾、覆寫磁碟，不問也不給反悔 |
| 提權 | `sudo`、`su` | 取得系統管理員權限，能改任何東西 |
| 網路 | `curl`、`wget`、`ssh`、`nc` | 能把檔案傳到外面、連到其他機器 |
| 危險操作 | `eval`、`chmod 777`、`git push --force`、`git reset --hard` | 執行任意程式碼、全開權限、強制覆蓋遠端程式碼、丟棄修改 |
| 機密檔案 | `.env`、`.ssh/`、`.aws/`、`*.key`、`*.pem` | API 金鑰、密碼、SSH 金鑰、雲端憑證、加密金鑰 |
| Shell 設定 | `.bashrc`、`.zshrc` | 被改了等於下次開終端機就自動執行惡意指令 |

除了清單裡列出來的，其他所有操作都會自動放行。

### 白名單派：預設全擋，只開放安全的

這條路適合「電腦裡有客戶資料、公司機密」或者「寧可多按幾次確認也不要出事」的人。跟黑名單邏輯相反：秘書什麼事情都要來請示老闆，我們再一項一項批准。

一樣跟 Claude 說：

```
幫我設定權限 settings.json，改成白名單模式：
- 只允許以下 Bash 指令自動執行：git、python3、node、pip install、pip3 install、npm install、ls、mkdir、mv、cp、rm、touch、cat、head、tail、grep、find、wc、sort、sed、echo、date、chmod
- 讀寫檔案也允許
- 其他所有 Bash 指令都要問我確認才能執行
```

設定完之後，清單裡的指令秘書可以自由使用，不在清單裡的每次都會跳確認。比如它想用 `curl` 去抓網頁資料，會停下來讓人看一眼，覺得 OK 再按 Allow。

好處：沒有驚喜，每個新指令都經過點頭。
壞處：偶爾會被不在清單裡的指令打斷工作流程。做的事比較多元的話，一開始會需要常常手動放行，再把常用的指令加進清單裡。

### 老實說：這些設定不是防彈衣

上面的黑名單和白名單都是「規勸」層級的防護，秘書會盡量遵守，但不是百分之百擋得住。舉個例子：你設了「禁止讀取 .env」，秘書用內建的 Read 工具去開 .env 確實會被擋，但如果它改用 `cat .env` 這種系統指令，就繞過去了。

這不是 bug，是設計上的限制。`settings.json` 管的是 Claude Code 自己的行為，不是作業系統層級的隔離。就像你告訴秘書「那個抽屜不要打開」，它會照做，但辦公室的鑰匙它還是有。

如果你電腦裡有真正不能外洩的東西（客戶個資、公司機密），最安全的做法是把那些檔案放在工作資料夾以外的地方，或是乾脆用另一台電腦。對大多數人來說，設好黑名單或白名單、再加上下面要介紹的 `.claudeignore`，日常使用已經夠安全了。

## .claudeignore：直接畫出禁區

不管選黑名單還是白名單，都建議設一個 `.claudeignore`。

`settings.json` 管的是「能做什麼動作」，`.claudeignore` 管的是「能看到什麼檔案」。兩層防護，邏輯不同。就算白名單裡只開放 `Read`，它還是能讀到工作資料夾裡所有檔案的內容，包括存了各種 API 金鑰的 `.env`。

`.claudeignore` 是一個純文字檔案，放在工作資料夾裡（就是開啟對話時選的那個資料夾根目錄）。檔案裡列的東西，Claude Code 完全看不到，不讀、不改、假裝不存在。

最簡單的建立方式：直接跟秘書說「幫我在目前資料夾建立 .claudeignore」，它會幫你建好。裡面這樣寫：

```
# 密碼和金鑰
.env
*密碼*
*password*

# 財務資料
報稅/
銀行對帳單/

# 大型檔案（不是安全問題，是省 token）
*.mp4
*.mov
*.zip
```

一行一個規則，`#` 開頭是註解。`*` 是萬用字元，`*密碼*` 表示檔名裡只要包含「密碼」兩個字就擋。資料夾後面加 `/` 表示整個資料夾都不讓它進。

有一件事要提醒：`.claudeignore` 只認檔名規則，不認檔案內容。一個叫做「機密資料.txt」的檔案，照上面的設定，AI 還是會去讀這個檔案的，因為檔名沒有命中任何規則。

## Hook：需要確認的就跳通知

Claude Code 有一個 Hook 機制，可以在特定事件發生時自動執行一段指令。我們拿來做自動通知用，當 AI 需要確認的時候，就跳一個通知，免得你滑手機忘了 AI 還在工作。

### 動手做：approve 時跳通知（Mac）

直接跟 Claude Code 說：

```
幫我在 settings.json 加一個 Hook，當你需要我確認的時候，用 osascript 在 Mac 跳一個通知提醒我，要有聲音
```

它會自動幫你把 Hook 設定寫進 `~/.claude/settings.json`，跟前面的權限設定合在一起，JSON 格式也不用你操心。

設好之後，每次 Claude Code 停下來等你按 approve，Mac 右上角就會跳一個通知氣泡，還會叮一聲。你在旁邊開著瀏覽器查資料、回 LINE 訊息，滑著手機的時候，聽到聲音就知道再切回去按確認就好。

### Windows 可以嗎？

Mac 的通知用的是 `osascript`，這個在 Windows 上沒有。Windows 要做到類似效果需要透過 PowerShell 的系統通知（toast notification），設定比較麻煩，而且不同版本的 Windows 行為不太一樣。Windows 用戶可以先跳過這段，後面章節教 Telegram 通知的時候會有跨平台的解法。

秘書的通行證設好了，敏感資料也鎖進保險箱，等待確認時還會跳提示音提醒回來處理。
